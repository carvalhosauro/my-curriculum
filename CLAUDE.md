# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Purpose

Career data repository for Gustavo Oliveira de Carvalho. Two use cases:

1. **Update career data** — when the user reports a new job, skill, or certification, update the correct files in `knowledge-base/perfil/`.
2. **Generate custom curriculum from a job URL** — fetch the page, extract requirements, map to profile + feitos, generate report + customized HTML curriculum.

## No Build System

Pure HTML/CSS/Markdown — no package manager or build tools. Curriculum files in `v0/` open directly in a browser. PDF export via browser print (`Ctrl+P`) with A4 paper and background graphics enabled.

## Architecture

```
knowledge-base/perfil/        ← structured career data (single source of truth)
  dados-pessoais.md           ← contact info, links
  experiencias.md             ← employment history
  habilidades.md              ← skills inventory with levels
  certificacoes.md            ← certifications
  formacao.md                 ← education + languages

knowledge-base/feitos.md      ← achievements with measurable impact
knowledge-base/QUANDO-ATUALIZAR.md  ← which file to update for each event type
knowledge-base/vagas/         ← job posting contexts (generated by agent)
knowledge-base/relatorios/    ← adherence reports (generated by agent)
v0/*.html                     ← curriculum versions (customized per job or profile)
```

## Skill

The custom Claude Code skill at `.cursor/skills/curriculo-agent-like/SKILL.md` defines both workflows in full detail. **Always read it before performing any career update or curriculum generation task.**

### Use case 1 — Update profile data

When user says "new job / new skill / new certification":
1. Check `knowledge-base/QUANDO-ATUALIZAR.md` to identify which files to update
2. Read those files to understand the existing format
3. Ask the user for the minimum required info
4. Update only with facts provided — never invent data

### Use case 2 — Generate curriculum from URL

When user shares a job URL:
1. `WebFetch(url)` → extract company, role, requirements, stack
2. Save job context to `knowledge-base/vagas/YYYY-MM-DD-empresa.md`
3. Read all `perfil/` files + `feitos.md`
4. Map requirements to experience (requisito → feito → aderência)
5. Generate report in `knowledge-base/relatorios/YYYY-MM-DD-empresa.md`
6. Generate customized HTML in `v0/YYYY-MM-DD-empresa.html` (base it on the closest existing HTML in `v0/`)

## Writing Rules for Bullets

- Structure: **context + action + measurable result**
- Impact verbs: `implementei`, `reduzi`, `estabilizei`, `instrumentei`, `automatizei`
- Every claim must trace to a feito or perfil entry — no fabrication, no seniority inflation

## Language

All documentation, perfil files, feitos, reports, and curriculum content are in **Portuguese**.
